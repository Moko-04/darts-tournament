# ダーツ ハウストーナメント運営アプリ

ダーツバーのハウストーナメント（**ダブルス主体**）運営ツール。参加者を入力するだけで、
グループ分け → 総当たり（ロビン）→ 決勝/ルーザートーナメント を自動生成する。
シングルスは当日募集の独立戦。店舗の台数を設定し、空き台に試合を割り当てて可視化する。

> このファイルは別PCでの開発引き継ぎ用。Claude Code が自動で読み込む前提で、
> 仕様・現状・実行方法・次の候補をまとめている。

## 技術構成
- **単一HTMLファイル `index.html`**。ビルド不要。React 18 + Babel standalone + Tailwind（すべてCDN）をブラウザで実行。
- 状態は **localStorage**（キー接頭辞 `darts_`）。バックエンドなし。
- UIは **ライト（白地・罫線）テーマ**。用語は「グループ」。

## 実行方法
- `node server.js` → 簡易静的サーバ http://localhost:4321
- または `index.html` をブラウザ（Chrome）で直接開く（CDN利用のためネット必要）
- Claude Code プレビュー: `.claude/launch.json` の `darts`（node server.js / port 4321）。
  デスクトップに移したらこのパスを各自の環境に合わせて修正すること。

## 画面（タブ）
1. **設定** — 店舗名 / 台数 / グループ数（2〜16, 既定4）。決勝進出数は自動（4組→2 / 5組以上→3, `advanceFor`）。下部に **データのエクスポート/インポート**（JSONバックアップ）。
2. **エントリー** — ペア枠（選手A・選手B、行追加式）＋シングル枠（相方募集、チップ一覧で可視化）。シングルチップは **ドラッグ&ドロップ**でペア枠スロット／別シングル／「＋ペアを追加」へ落として相方指定。合計人数表示。
3. **組分け** — チームをグループにランダム配置＋ドラッグで手動移動。
4. **ロビン表** — グループごとの総当たり表。上半分のマスに直接レッグ数入力、下半分は自動ミラー。`勝敗 / Legs(得失) / 順位` 列。同率は直接対決(H2H)→レッグ差。
5. **決勝トナメ / ルーザー** — ロビン順位から自動シード（クロスシードで同組を別山へ）。名前クリックで勝者確定→次戦へ進行。
6. **シングルス** — 当日募集の個人戦。独立した名簿→勝ち抜きトーナメント。
7. **ボード状況** — 進行可能な試合を空き台に自動割当して可視化。

## データモデル（localStorage / `darts_*`）
- `cfg`: `{store, boards, groups}`
- `pairs`: `[{id,a,b}]` 固定ペア＋確定したシングル組
- `singles`: `[name]` 相方募集中
- `teams`: `[{id,name,members,solo}]` 振り分け後の確定チーム（**ダブルス=2名で1チーム**）
- `groups`: `[[teamId,...],...]`
- `rr`: `{groupIdx: {"a_b": {a,b,sa,sb,winner}}}`（a<b 正規化）
- `brk`: `{winners, losers, singles}` 各ブラケット `{rounds:[[match,...],...]}`
- `assign`: `{globalMatchId: boardNo}`
- `singlesNames`: シングルス用名簿テキスト

## 主要関数（index.html 内）
- `advanceFor(g)` 決勝進出数（固定ロジック）
- `entriesToTeams(pairs, singles)` エントリー→チーム（残りシングルは自動でペア化）
- `distribute(teams, groupCount)` ランダム配分
- `standings(gi)` 順位（勝数 → 直接対決H2H → レッグ差）
- `buildBracket(seeds)` / `seedOrder(n)` / `resolveSlot` / `matchWinner` トーナメント
- `exportData()` / `importData(file)` JSONバックアップ（キー一覧は `STATE_KEYS`）

## 注意点・既知の仕様
- グループ数変更で即再振り分け（`changeGroupCount`）。設定と `groups` 配列がズレると全タブ上部に警告バナー＋ワンクリック修正。
- 4グループには最低8チーム（各組2チーム）必要。少人数向けに下限は **2グループ**まで。
- localStorage は端末ごと。**別端末へのデータ移動はエクスポート/インポート**で行う（コードはGit、データはJSON）。
- Claude Code のプレビューはユーザーの実ブラウザと localStorage が共有される場合がある。検証で `localStorage.setItem('darts_*')` するとユーザーの実データを上書きしうる → なるべくUI操作で検証し、テストデータは最後にクリアすること。
